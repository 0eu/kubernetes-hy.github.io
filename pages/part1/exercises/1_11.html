Exercise 1.11: Persisting data

Create both a *PersistentVolume* and *PersistentVolumeClaim* and alter the *Deployment*. As *PersistentVolume* is often maintained by cluster administrators rather than developers and are not application specific you should keep the definition for that separated. In the end the two pods should share a persistent volume between the two applications. Save the number of requests to ping / pong application into a file in the volume and output it with the timestamp and hash when sending a request to our main application. So the browser should display the following when accessing the main application: ```plaintext 2020-03-30T12:15:17.705Z: 8523ecb1-c716-4cb6-a044-b9e83bb98e43. Ping / Pongs: 3 ```
